# 个人网站技术方案

## 技术栈选型

| 技术 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 (App Router) | 内置 SSG/SSR，SEO 友好，服务端组件减少客户端 JS |
| 语言 | TypeScript | 类型安全，提升可维护性 |
| 样式 | Tailwind CSS v4 | 原子化 CSS，原生支持暗黑模式 |
| 动画 | Framer Motion | React 生态最成熟的动画库 |
| 内容管理 | Contentlayer | Markdown/MDX 转类型安全数据，零运行时开销 |
| 国际化 | next-intl | 专为 App Router 设计的 i18n 方案 |
| UI 组件 | shadcn/ui | 高度可定制，按需引入 |
| 部署 | Vercel | 与 Next.js 深度集成，自动 CI/CD |

## 页面结构

- **首页** — Hero 区域 + 个人介绍 + 精选项目 + 最新博客
- **关于我** — 个人背景、经历时间线
- **技能展示** — 技能雷达图、技术栈分类
- **项目作品集** — 项目卡片、分类筛选、详情页
- **博客** — 文章列表、分类/标签筛选、详情页（MDX 渲染 + 代码高亮）
- **联系方式** — 表单 + 社交链接

## 组件架构

采用三层分层架构：

```
Pages (页面) → Sections (区块) → Components (组件)
```

服务端组件负责 SEO 和首屏渲染，客户端组件（`'use client'`）处理主题切换、动画、表单等交互。

## 数据流

```
MDX 文件 → Contentlayer 构建时解析 → 类型安全的数据工具函数 → Server Components 直接获取 → SSG 静态输出
```
